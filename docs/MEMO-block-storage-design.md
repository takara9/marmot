# ネットワークブロックストレージの設計メモ


## ボリュームコントローラーに機能を追加

### 前提条件
- 機能はロジカルボリュームに限定
- 仮想マシンには、iSCSIのイニシエーターIDがセットされている。


### ブロックストレージ作成のプロセス

#### ターゲット側の設定
- APIでリモートアクセスのブロックストレージ作成共有を受ける
- ロジカルボリュームを作成する
- ターゲットのバックストアーを作成
- ターゲットを作成
- ターゲットとバックストアを対応付け、LUNを作成
- アクセス許可先のイニシエーターのIQNを設定
- CHAPを有効化
- イニシエーターのIQNに対して、認証ユーザーをセット
- イニシエーターのIQNに対して、パスワードを設定
- 設定を保存


#### イニシエーターの設定
- ドメインを作成する時のXMLファイルに、DISK設定を追加
    - 認証のためのシークレットを指定
    - iSCSIのターゲット名＋LUN番号をセット
    - iSCSIターゲットのIPドレスポート
    - 自己のiSCSIイニシエーター名をセット
